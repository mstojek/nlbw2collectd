# Architecture

## Design Pattern
The project implements an **Exporter / Integration pattern** bridging two disconnected tools within OpenWrt:
- **Source:** `nlbwmon` (Network Bandwidth Monitor)
- **Destination:** `collectd` (System statistics collection daemon)

## Components & Data Flow
1. **Data Collection (Lua Plugin):**
   A Lua script runs inside `collectd-mod-lua`. It connects to `ubus` (OpenWrt's microbus architecture) and periodically queries `nlbwmon` for network traffic data.
2. **Metrics Emission:**
   The Lua script formats the received data into collectd values and dispatches them via collectd's internal API.
3. **Visualization (LuCI JS Definition):**
   A JavaScript definition instructs `luci-app-statistics` on how to read the RRD files (generated by collectd) and render graphs in the OpenWrt web interface.

## Entry Points
- `nlbw2collectd.lua` via collectd's lua plugin module loading.
- OpenWrt Makefile installation hooks (`postinst`/`postrm`).
